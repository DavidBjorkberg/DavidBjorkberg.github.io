RTS Movement

(Work In Progress)

This is a boids based avoidance system with alterations to minimize overlap between pawns and to stop movement outside of the navmesh.
The pawn gets an initial direction towards its destination and when it gets within a certain range of another pawn, it checks that pawns position & direction and modifies its own direction to avoid it.

To make sure it doesn't leave the navmesh it queries the navmesh for its current polygon and the polygon it would be next frame with its current direction, if it is not allowed to move to that polygon, it counts it as terrain and adjusts its direction accordingly.

The performance is currently too poor to release it to FAB.
To fix this I am doing three things:
1. Creating an Actor partition, to make sure the pawns only queries pawns that are reasonably close. Reducing CPU cost but increasing memory usage.
2. The cost of computing the new direction when colliding with a pawn can get quite high (especially when its more than one collision), but since its all vector math, I can move this to the GPU which can compute it much faster.
3. Implementing options for the user to have much cheaper but less precise movement.

Unreal Engine provides its own avoidance behaviour, however this is too floaty and imprecise to be used in a game where the pawn movement matters heavily. It is more suitable for crowd behaviour that is not central to the gameplay.
