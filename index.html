    DirectX Game - Team project

    • Worked in a team of 11
    • Responsible for all AI
    • DirectX
    • C++
    • A* Pathfinding

    This is an action car game, with a focus on fast-paced action with a high amount of enemies around the player at all times.
    Worked with ten other people to create a game from the ground up in DirectX over four months.
    Here I was responsible for all AI, with a focus on performance.
    For the large number of enemies to be able to exist at the same without any lag, a lot of effort had to be spent on making the AI cost as little to run as possible.

    Techniques I implemented to achieve this was; computing a less accurate path for enemies that were far away, calculating one path for a group of enemies, instead of one path for each enemy, etc.

    This project can be found on my Github as "cplusplus_DirectX_Game".

    Unity FPS (Work in progress)

    • Unity
    • C#
    • Custom mesh
    • State-based AI system

    A First-person-shooter game built in Unity.
    When the right mouse button is held down, three things happen:
    • Players bullets bounce off the first wall they hit
    • A custom mesh ray is shot from the players gun, to help aim the bounce direction
    • A camera is spawned at an area close to the bounce position, which can see the the ray from the bounce position to the rays end position.
    This mechanic can be seen in the GIF above.

    Textures were retrieved from the Unity Asset Store.
    Environment: https://assetstore.unity.com/packages/3d/environments/3d-free-modular-kit-85732
    Crosshair: https://assetstore.unity.com/packages/2d/gui/icons/crosshairs-plus-139902
    This project can be found on my Github as "Unity_FPS-Work-in-progress".

    Thingy features

  • Editor:
    • Ladder that snaps and extends to a roof and a floor
    • Gameobjects snap to floor/ceiling
    • Moving platforms that follows nodes, nodes are managed through buttons in the inspector
    • Gizmos are used to draw the moving platforms path, bouncepad direction, turret direction, etc.
    • Button that can be connected to other activatable gameobjects with a button in the inspector, using UnityEvents.
    • Bouncepad with an adjustable strength slider in inspector
    • Portal that can be linked with other portals through inspector
    • Disappearing / appearing platforms that can be linked to button
    • Remote door that can be linked to button
    • Box which drops a weapon assigned through button in inspector (Random if multiple)
    • Static wall turret and wall turret that aims after a player

  • Support for keyboard and controller, up to 4 players

  • Score screen: Each player grows according to their accumulated score, largest player is the winner
