Event Queue

Unreal Engine plugin that allows gameplay logic to be queued, stalled, removed, or reacted to.
Enabling latent and dynamic gameplay in an easy and clear way without relying on callbacks.

In developing a turn-based game, I created a system to make gameplay more dynamic and manageable

Initially, I used callbacks and delegates for injecting item-specific logic into the gameplay. for example if the player had a lifesteal item, i wanted the player to gain health and play a heal animation after dealing damage before continuing with the turn.This approach, however, led to complicated, hard-to-maintain code

To solve this i created a plugin, where you can encapsulate a piece of gameplay logic in an "Event".
For example: to create an Attack event, you would simply create an class inheriting from the Event class, place your attack logic inside it and add the event to the Event Queue.
When the queue reaches the Attack event, it executes its logic and then moves to the next Event in the queue.

All you'd have to do to implement a lifesteal item now is to:
1. Subscribe to the delegate that notifies when a new event triggers.
2. When the delegate triggers, insert a heal event as the next Event in the queue.

Additionally, you can prevent the queue from proceeding to the next Event by adding a "Blocker." As long as an event has at least one blocker, the queue won't move on to the subsequent event.
With these components you can easily create dynamic & latent gameplay that scales well with the project.
