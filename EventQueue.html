Event Queue

Plugin that allows gameplay logic to be queued up and latently executed.

When developing a turn-based game, I initially used callbacks and delegates to insert item-specific logic into the gameplay. This approach led to complex, hard-to-maintain code.
To address this, I created a plugin where gameplay logic is encapsulated in "Events."

For example, to handle an attack, you create an Attack Event class, place the logic inside it, and add it to the Event Queue. When this Event triggers, its logic executes before moving to the next event in the queue.
All you'd have to do to implement a lifesteal item now is to:
1. Subscribe to the delegate that notifies when a new event triggers.
2. When the delegate triggers, insert a heal event as the next Event in the queue.

Additionally, the system allows adding "Blockers" to prevent the queue from proceeding until certain conditions are met.
With these components you can easily create dynamic & latent gameplay that scales well with the project.
